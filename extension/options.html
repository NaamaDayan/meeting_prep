Send briefing (Gmail)

“Send briefing” uses a separate Google sign-in. The OAuth client type Chrome extension does not register redirect URIs for that flow, which causes redirect_uri_mismatch. Create a second client: OAuth 2.0 → Web application, then under Authorized redirect URIs add the value below exactly (copy/paste).

From Google Cloud → Credentials → your Web application OAuth client (not the Chrome extension client). Same Google Cloud project as the extension. Under APIs & Services → OAuth consent screen → Scopes, include …/auth/gmail.send and …/auth/userinfo.email (Send Briefing needs both so the app can read the sending address; gmail.send alone cannot call Gmail’s profile API).
